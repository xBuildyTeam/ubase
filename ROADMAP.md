# uBase Roadmap

This document outlines the planned direction for the uBase standard and marketplace. Items are organized by phase — not by strict timeline. Community input shapes priorities.

---

## ✅ Phase 1 — Foundation (Shipped)

- [x] `.ubase` zip format standard defined
- [x] `meta.json`, `prompt.md`, `README.md` structure
- [x] Marketplace at ubase.tech (browse, install, submit)
- [x] Packet Builder — generate packets from plain-English descriptions
- [x] AI-guided installation via Base44 builder
- [x] Packet versioning and draft/review/published workflow
- [x] Stripe-powered paid packets
- [x] Star ratings and reviews

---

## 🔄 Phase 2 — Community (In Progress)

- [ ] GitHub-based packet contribution workflow (this repo)
- [ ] `CONTRIBUTING.md` and review process
- [ ] Community contributor profiles on ubase.tech
- [ ] Packet dependency declarations in `meta.json`
- [ ] Packet update notifications for installed packets
- [ ] CLI tool: `npx ubase install <packet-name>`

---

## 🔮 Phase 3 — Ecosystem

- [ ] Packet collections / bundles (install multiple at once)
- [ ] Packet analytics dashboard for authors (installs, views, ratings)
- [ ] Verified author badges
- [ ] Sponsored/featured packet slots
- [ ] Auto-sync: GitHub PR merge → auto-publish to marketplace
- [ ] Packet forking UI on ubase.tech
- [ ] Base44 native integration listing

---

## 💡 Future Ideas

- Multi-platform support (beyond Base44)
- Packet testing framework
- AI-powered packet discovery ("find me a packet that does X")
- Enterprise private packet registries
